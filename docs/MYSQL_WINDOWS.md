# Запуск MySQL на Windows - Подробная инструкция

## Быстрый запуск

### Через службы Windows (самый простой способ)

1. Нажмите `Win + R`
2. Введите `services.msc` и нажмите Enter
3. В списке служб найдите **MySQL80** (или просто MySQL)
4. Правой кнопкой мыши → **Запустить** (Start)
5. Готово! MySQL запущен

### Установка автозапуска

Чтобы MySQL запускался автоматически при включении компьютера:
1. В службах Windows найдите MySQL80
2. Правой кнопкой → **Свойства**
3. В поле "Тип запуска" выберите **Автоматически**
4. Нажмите **ОК**

## Альтернативные способы запуска

### Через командную строку (PowerShell/CMD от администратора)

```powershell
# Запустить MySQL
net start MySQL80

# Проверить статус
sc query MySQL80

# Остановить MySQL (если нужно)
net stop MySQL80
```

### Через MySQL Command Line Client

1. Откройте меню Пуск
2. Найдите "MySQL" → "MySQL Command Line Client"
3. Запустите приложение
4. Введите пароль root (тот, который указали при установке)
5. Если появилось приглашение `mysql>`, значит MySQL работает

## Проверка работы MySQL

### Способ 1: Командная строка
```bash
# Откройте PowerShell или CMD
mysql -u root -p

# Введите пароль
# Если видите приглашение mysql> - всё работает!
```

### Способ 2: Проверка версии
```bash
mysql --version
```
Должна отобразиться версия MySQL, например: `mysql  Ver 8.0.xx`

### Способ 3: Подключение через Workbench
1. Откройте MySQL Workbench
2. Создайте новое подключение:
   - Host: `localhost` или `127.0.0.1`
   - Port: `3306`
   - Username: `root`
   - Password: ваш пароль root
3. Если подключение успешно - MySQL работает

## Устранение проблем

### Проблема: "MySQL80" не найдена в службах

**Решение:**
1. Убедитесь, что MySQL правильно установлен
2. Проверьте установку через "MySQL Installer"
3. Если службы нет, переустановите MySQL и убедитесь, что выбрали установку как службы Windows

### Проблема: Ошибка "Access denied" при входе

**Решение:**
1. Проверьте правильность пароля root
2. Если забыли пароль:
   ```bash
   # Остановите MySQL
   net stop MySQL80
   
   # Запустите MySQL в безопасном режиме (нужно настроить my.ini)
   # Или используйте MySQL Installer для сброса пароля
   ```

### Проблема: Порт 3306 уже используется

**Решение:**
1. Проверьте, не запущен ли уже MySQL:
   ```powershell
   netstat -ano | findstr :3306
   ```
2. Если процесс запущен, но служба не работает, завершите процесс
3. Или измените порт в настройках MySQL (файл `my.ini`)

### Проблема: Команда `mysql` не найдена

**Решение:**
1. MySQL может быть не добавлен в PATH
2. Найдите папку установки MySQL (обычно `C:\Program Files\MySQL\MySQL Server 8.0\bin`)
3. Добавьте её в переменную PATH:
   - Панель управления → Система → Дополнительные параметры системы
   - Переменные среды → Path → Изменить
   - Добавьте путь к папке `bin`

Или используйте полный путь:
```powershell
"C:\Program Files\MySQL\MySQL Server 8.0\bin\mysql.exe" -u root -p
```

## Полезные команды

```sql
-- Войти в MySQL
mysql -u root -p

-- Показать все базы данных
SHOW DATABASES;

-- Создать базу данных
CREATE DATABASE cookie_shop;

-- Использовать базу данных
USE cookie_shop;

-- Показать таблицы
SHOW TABLES;

-- Выйти из MySQL
EXIT;
```

## Где находится конфигурационный файл

На Windows конфигурационный файл MySQL обычно находится здесь:
```
C:\ProgramData\MySQL\MySQL Server 8.0\my.ini
```

Для изменения настроек (порт, пароль и т.д.) отредактируйте этот файл и перезапустите службу MySQL.

## Настройка для проекта Cookie Shop

После запуска MySQL выполните:

```bash
# Войдите в MySQL
mysql -u root -p

# Или используйте полный путь, если MySQL не в PATH
"C:\Program Files\MySQL\MySQL Server 8.0\bin\mysql.exe" -u root -p
```

Затем выполните скрипт создания базы данных:
```bash
mysql -u root -p < database/schema.sql
```

Или вручную в MySQL консоли:
```sql
SOURCE D:/cookie-shop/database/schema.sql;
```

**Примечание:** Используйте правильный путь к файлу schema.sql на вашем компьютере.
